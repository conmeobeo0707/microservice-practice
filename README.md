# 🧩 Microservices Architecture Demo

## 📌 Overview
This project is a microservices-based system built for learning and experimentation purposes. It demonstrates how multiple services can communicate and coordinate using both synchronous and asynchronous patterns.

> **⚠️ Note:** This project is based on tutorials and has been implemented as a hands-on learning exercise to understand microservices architecture and related technologies.

---

## 🚀 Tech Stack

### 🧱 Architecture
- **Microservices Architecture**
- **Service-to-service communication patterns**

### 🔧 Backend & Infrastructure
- **.NET** (ASP.NET Core Web API)
- **Docker** – containerization
- **Kubernetes** – container orchestration
- **RabbitMQ** – asynchronous messaging (event-driven communication)

### 🔗 Communication
- **HTTP REST API** – synchronous communication
- **gRPC** – high-performance service-to-service communication
- **RabbitMQ** (Message Broker) – asynchronous communication

---

## ⚙️ Key Concepts Demonstrated
- 📦 **Service separation** and independent deployment
- 🔄 **Asynchronous messaging** with RabbitMQ
- 🔗 **Synchronous communication** using HTTP & gRPC
- 🐳 **Containerization** with Docker
- ☸️ **Deployment and orchestration** using Kubernetes
